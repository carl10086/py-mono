# Py312 Skill - Python 3.12+ 严格代码质量规范

## 核心特点

1. **函数长度严格控制**（≤30行）
2. **标准中文注释**（文档字符串 + 关键逻辑）
3. **Pyright 严格模式**（零容忍类型检查）
4. **简单就是美**（避免过度设计）
5. **不确定时主动沟通**（关键决策前询问）

## 关键规则

### 类型检查（严格模式）
- 零 `Any` 容忍
- 完整泛型参数化：`list[str]` 而非 `list`
- 使用 `@override` 装饰器
- 禁止隐式可选参数

### 代码质量命令
```bash
# 格式化与风格检查
ruff check . --fix
ruff format .

# 严格类型检查（强制！）
pyright .
```

### 设计哲学
- **简单 > 复杂**
- **显式 > 隐式**
- **拆分长函数**
- **中文注释说明"为什么"**

## 适用场景

- 编写新的 Python 3.12+ 代码
- 重构遗留代码
- 代码审查前的自检
- 学习现代 Python 类型系统

## 快速开始

```bash
# 一键检查代码质量
.claude/skills/py312/scripts/check_py312.sh .
```

## 与现有工具的关系

- **poe typecheck** ≈ **pyright**：前者是任务别名，后者是实际工具
- 推荐直接使用 `pyright .` 更明确
- Ruff 负责风格，Pyright 负责类型安全（双保险）
